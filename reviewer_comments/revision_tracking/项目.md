# 项目工作规范（第三版起执行）

## 1. 同步修改原则

- 后续所有代码研究层面的修改必须同步落实到主工作文件夹和四个 Jupyter Notebook 中，不只修改临时脚本或验证副本。
- 主 notebook 为：
  - `01_Parametric_Simulation_Database_Construction.ipynb`
  - `02_SRC_Sensitivity_and_Variable_Selection.ipynb`
  - `03_ML_Model_Training_and_Evaluation.ipynb`
  - `04_EUI_OCEI_Coupling_and_Carbon_Analysis.ipynb`
- 若新增图件、表格或审稿证据，应同时保存到对应的 `outputs_step*/` 目录，并在 notebook 中保留可复现生成逻辑。
- 若新增工具脚本，应放入 `tools/`，并在 notebook 或项目说明中写明调用方式、输入、输出和适用场景。

## 2. Notebook 说明 cell 规范

- 每个代码 cell 前必须有中英文说明 cell。
- 说明应解释当前 cell 的实际工作，包括研究目的、输入依赖、核心计算、输出文件和复现注意事项。
- 不再在每个 cell 中重复强调“回复审稿意见”。评委应通过具体计算、图表和输出文件自然理解该 cell 对研究问题的支撑作用。
- 中文说明应准确、正式、便于复现；英文说明应与中文信息一致，不使用夸大表述。
- 变量名、文件名、单位、公式和代码标识符保持原文，例如 `EUI`, `OCEI`, `kgCO2e/(m2·a)`, `random_state=42`。

## 3. 图件质量与图文一致性 QA

所有论文图件在提交前必须完成以下检查：

1. **无图文重合**：标题、轴标签、刻度、图例、数值标注、面板标号、箭头和说明文字不得遮挡数据或相互覆盖。
2. **语义准确**：图题、图例、变量名、单位和箭头方向必须与代码计算逻辑一致；不得用装饰性元素替代真实研究含义。
3. **颜色友好**：避免仅用红绿区分关键类别；优先使用蓝、橙、灰、绿色等色盲友好组合，并保证灰度打印仍可辨读。
4. **版面稳定**：多面板图使用明确的网格、间距和留白；长标签需折行，不得靠旋转文字强行塞入。
5. **输出格式**：关键图件至少保存 PNG 和 SVG；论文排版优先使用 SVG/PDF，快速核查使用 PNG。
6. **字体检查**：中文图件优先使用 `Microsoft YaHei` 或 `SimHei`；若字体回退，应重新打开导出图确认中文未缺字。
7. **尺寸检查**：单幅图不应出现异常超长画布；示意图应控制为横向或合理多面板布局，便于论文插入和评审阅读。
8. **源数据可追溯**：定量图应能追溯到 notebook 计算过程、CSV 输出或脚本中的明确变量。

## 5. 本地验证原则

- 修改 notebook 或脚本后，必须在本地 Python/Jupyter 环境运行验证。
- 快速验证可使用：

```powershell
$env:EUI_FAST_MODE = "1"
$env:EUI_N_SAMPLES = "120"
$env:EUI_RUN_ENERGYPLUS = "0"
```

- 正式全量仿真仍需：

```powershell
$env:EUI_FAST_MODE = "0"
$env:EUI_N_SAMPLES = "20000"
$env:EUI_RUN_ENERGYPLUS = "1"
```

- 当前完整 EnergyPlus 全量样本仍需长时运行，不能把调试样本的模型表现等同于完整论文结论。
