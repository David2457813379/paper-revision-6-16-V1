# 审稿意见重要性与代码修订审计表（2026-06-15）

## 1. 审计原则

本表依据 MDPI 官网截图中的审稿意见、项目内已整理的 Reviewer Comments Master Checklist，以及四个 Notebook 的最新代码状态编制。排序原则不是按审稿人编号，而是按“若不充分回应是否会阻碍接收”的科研风险分级：

- **P0：阻断性关键问题**。直接影响研究结论可信度、可复现性或碳核算有效性，必须在代码、论文正文和回复信中同时闭环。
- **P1：高优先级方法问题**。影响方法透明度、模型解释和审稿人对结果稳健性的信任，必须提供计算证据和正文说明。
- **P2：中优先级论文结构问题**。影响论证完整性和读者理解，需要正文重写或补充讨论。
- **P3：编辑与呈现问题**。不一定改变结论，但会影响出版质量和审稿体验，需要全文统一处理。

## 3. 四个 Notebook 的修改路线与证据

### 3.1 Notebook 01：参数化仿真数据库构建

**目标定位**：解决样本基础、可复现性和 Sim-to-Real 边界问题。

已完成：

- 每个代码 Cell 前已新增中文和英文双语解释。
- `CONFIG["n_samples"]` 改为读取 `EUI_N_SAMPLES`，正式模式默认 20,000，快速验证可设为 120。
- `CONFIG["run_energyplus"]` 改为读取 `EUI_RUN_ENERGYPLUS`，默认不误触发长时间仿真。
- 保留 `source_lhs_index`，修复筛选前后覆盖图原本可能因 reset index 产生的掩码错位。
- 新增逐变量筛选前后分布差异表：KS statistic、KS p-value、Jensen-Shannon distance。
- 保留真实 EUI 对标图，用于说明代理数据与北京酒店实测数据的范围关系。

仍需论文正文落实：

- 明确模拟 EUI 均值约高于实测均值 14% 的事实，不应只强调范围重叠。
- 说明 77% 剔除率来自多个几何变量独立采样后的组合可行性约束，不是按 EUI 输出筛选。
- 若无法取得北京酒店详细面积分配调研数据，应在局限性中说明数据不可得和采用规范边界的理由。

### 3.2 Notebook 02：SRC 敏感性与关键变量选择

**目标定位**：解决 SRC 线性假设、非线性验证和 18 变量截断依据。

已完成：

- 每个代码 Cell 前已新增中文和英文双语解释。
- 增加 `EUI_FAST_MODE`，快速验证时降低 bootstrap 和 SHAP 训练成本。
- SHAP 分析以 XGBoost 作为非线性解释补充，输出 SRC-SHAP 排序对比。
- 变量筛选新增三条证据线：SRC 碎石图、累计 `|SRC|` 贡献曲线、交叉验证 R² 随变量数量变化曲线。

仍需论文正文落实：

- 不把 SRC 说成能完整刻画非线性和交互作用，而应表述为“一阶筛选工具”。
- 说明选择 SRC 的理由：计算成本低、边际效应直观、适合大规模 EnergyPlus 样本的一阶解释；SHAP 用于非线性稳健性验证。
- 以定量结果说明 18 变量不是任意截断，而是累计贡献、CV 平台和碎石图共同支持。

### 3.3 Notebook 03：机器学习模型训练与评估

**目标定位**：解决高 R² 可信度、超参数可复现性和非核心变量固定问题。

已完成：

- 每个代码 Cell 前已新增中文和英文双语解释。
- 新增 `EUI_FAST_MODE`、`INNER_CV` 和 `SEARCH_N_ITER`，快速验证与正式搜索分离。
- `fit_and_compare_models` 返回 fitted models、search objects 和 best params，避免调参结果只存在内存中。
- `model_hyperparameters.csv` 输出全部模型最终超参数、搜索方式和搜索空间。
- 使用 Pipeline 组织 imputer、scaler 和 model，避免在 train/test split 前泄漏预处理统计量。
- 新增 18 变量与全变量模型对比，回应非核心变量固定是否人为提高性能。

仍需论文正文落实：

- 所有 “prediction accuracy” 应谨慎改为 “surrogate fidelity on EnergyPlus simulations” 或等价表述。
- 报告前五个模型最终超参数，完整表放补充材料。
- 解释 KNN、RF 在结构化仿真数据上弱于多项式/正则线性模型的原因，避免被认为异常或错误。

### 3.4 Notebook 04：EUI-OCEI 耦合与碳排放分析

**目标定位**：解决排放因子来源、情景敏感性、能源载体贡献和图表冗余。

已完成：

- 每个代码 Cell 前已新增中文和英文双语解释。
- 排放因子敏感性分析已移至 OCEI 数据构建之后，避免执行顺序错误。
- 对能源列做数值化和缺失值处理，OCEI 建模前过滤无效 target。
- 敏感性分析输出 OCEI 均值、EUI-OCEI 相关性、Top-10% 排名重叠率以及各能源载体碳贡献份额。
- 合并绝对值与归一化碳贡献图，生成双视图图件。

仍需论文正文落实：

- 清晰解释区域供冷 0.16 kgCO₂e/kWh 的推导，即基于电力因子和 COP 反算。
- 讨论电网脱碳后电力碳贡献下降、天然气/DHW 相对贡献上升，EUI-OCEI 耦合结构会随时间变化。
- 区分 OCEI 绝对值对因子敏感和排序/相关结构相对稳健这两个层面。

3. **长期增强路线**：若取得真实酒店账单或调研数据，增加外部校准/验证章节；若后续面向政策外推，按酒店星级或运营类型分层建模。

## 6. 不应在回复信中过度承诺的事项

- 不应声称模型已经验证了真实酒店预测精度；只能声称已用公开北京酒店实测范围进行了 benchmark，并明确仍存在 Sim-to-Real Gap。
- 不应声称 77% 剔除率没有任何影响；应说筛选基于几何可行性而非输出变量，并已用分布差异诊断量化其影响。
- 不应声称 SRC 能替代 Sobol 或完整非线性全局敏感性分析；应说 SHAP 补充验证支持本研究的一阶变量排序。
- 不应声称排放因子不会影响 OCEI；应区分绝对值敏感和排序结构稳健。
